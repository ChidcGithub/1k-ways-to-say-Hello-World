# #0108 · C++ X-Macro Preprocessor

## Description

A classic-style Hello World built with the X-macro preprocessor idiom. A separate `greeting_parts.def` file contains four invocations of an undefined macro `PART`, each carrying one fragment of the greeting. The C++ source defines `PART(x)` to mean `s+=x;`, then `#include`s the `.def` file directly inside the body of a `build()` function, after which the macro is `#undef`'d. The preprocessor pastes the four `PART(...)` lines in place, each becoming one `s+=...;` statement, so the function returns the concatenated string `"Hello, World!"`. This is the X-macro pattern: a list-shaped data file expanded differently by whoever includes it. It differs from the plain macro entry (#0107) by externalising the data into a reusable list file.

## Source Code

`greeting_parts.def`:

```text
PART("Hello")
PART(", ")
PART("World")
PART("!")
```

`main.cpp`:

```cpp
#include <iostream>
#include <string>
std::string build(){
    std::string s;
#define PART(x) s+=x;
#include "greeting_parts.def"
#undef PART
    return s;
}
int main(){ std::cout<<build()<<std::endl; }
```
